既然是面向 **Claude Code** 的实战开发路线图，我们需要按照**“依赖关系”**和**“最小可行性产品 (MVP)”**的逻辑来排布。

这份路线图的核心策略是：**先通底层逻辑，再还原视觉界面，最后补全管理细节。**

---

## 🛠️ 技术开发路线图 (Implementation Roadmap)

### 第一阶段：基础设施与数据建模 (Week 1)

**目标：** 在 Claude Code 中建立项目的“骨架”，确保数据库和环境就绪。

1.  **环境初始化：** 配置后端框架（如 NestJS）与数据库连接。
2.  **数据建模：** 按照 PRD 5.1 节定义的 Schema 创建所有 Migration 文件。
3.  **身份认证：** 实现微信登录逻辑，区分 `User` 和 `Merchant` 角色。
4.  **商户入驻：** 实现基础的商户资料 CRUD 接口（为后续展示做准备）。

> **Claude 指令建议：** “请基于 PRD 2.2 节的数据字典，生成所有 MySQL 迁移文件和基础的 CRUD Service。”

---

### 第二阶段：核心交易引擎 (Week 2-3)

**目标：** 让“钱”能流转起来。

1.  **支付对接：** 集成微信支付 V3。实现下单（预扣库存）、支付回调、订单状态更新。
2.  **库存安全：** 引入 Redis 锁，确保在高并发购券场景下不超卖。
3.  **卡包逻辑：** 实现“我的券包”列表接口，支持按状态过滤（待使用/已使用）。
4.  **退款逻辑：** 编写退款服务，支持用户手动申请和系统自动检查。

---

### 第三阶段：UI 还原与前端连接 (Week 4)

**目标：** 接入 Stitch 的产出，让系统“有脸面”。

1.  **MCP 挂载：** 将 Stitch 生成的 UI 代码放入 `/stitch-output`。
2.  **前端还原：** 启动 Claude Code，逐个页面进行 Uniapp 1:1 还原（首页、详情页、券包页）。
3.  **数据绑定：** 将前端页面与第二阶段完成的 API 进行联调。
4.  **动态出码：** 实现小程序端 30s 自动刷新的核销二维码生成算法。

> **Claude 指令建议：** “请读取 MCP 挂载的 `/stitch-output/wallet.vue`，在 `/src/pages` 中还原该页面，并绑定 `GET /api/v1/orders/qrcode` 接口。”

---

### 第四阶段：核销与安全闭环 (Week 5)

**目标：** 完成线下最重要的操作节点。

1.  **商户扫码：** 实现商户端调起微信扫码器，并解析动态码。
2.  **核销校验：** 编写核心核销函数，校验 `merchant_scope` 权限及二维码时效性。
3.  **状态反馈：** 完成核销后的实时台账更新，并触发商户端的成功/失败反馈界面。

---

### 第五阶段：内容与导购模块 (Week 6)

**目标：** 补全“商场”氛围感功能。

1.  **CMS 开发：** 实现新闻资讯的发布、分类、及 Banner 管理。
2.  **内容联动：** 开发“商户-券”联动逻辑（详情页自动加载关联券）。
3.  **搜索增强：** 实现商户与活动的模糊搜索及行业过滤。

---

### 第六阶段：财务与对账系统 (Week 7)

**目标：** 解决甲方财务的痛点。

1.  **报表引擎：** 实现月度核销流水聚合查询。
2.  **快照生成：** 实现结算单生成逻辑，并锁定对应周期的订单数据。
3.  **导出功能：** 集成 Excel 导出库，生成符合甲方财务要求的对账单。

---

### 第七阶段：加固、压测与上线 (Week 8)

**目标：** 最后的防线。

1.  **压力测试：** 模拟 5.1 期间的突发流量，优化数据库索引。
2.  **自动化退款：** 部署 5.4 凌晨执行的自动退款 Cron Job。
3.  **提审上线：** 提交微信小程序审核（预留 3-5 天）。
