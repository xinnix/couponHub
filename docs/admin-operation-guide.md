# 管理端操作手册

> **面向商场运营团队的功能使用指南**
>
> 本文档提供管理端各功能模块的详细操作步骤和使用说明。

---

## 一、系统概述

### 1.1 系统定位

**汉都天地商场数字营销与结算系统** 管理端(Web后台)是整个系统的运营管理中心,为商场运营团队提供全方位的业务管理、财务结算和数据监控功能。

### 1.2 核心价值

- **业务管理**: 商户入驻、优惠券发行、订单监控全流程管理
- **财务结算**: 自动生成结算报表,确保商户结算透明可信
- **数据运营**: 实时数据监控,助力运营决策
- **权限管控**: 完善的RBAC权限体系,保障数据安全

### 1.3 登录信息

**访问地址**: http://localhost:5173 (开发环境)

**测试账号**:
- 超级管理员: `superadmin@example.com` / `password123`
- 普通管理员: 需由超级管理员创建

## 三、功能模块详解

### 3.1 仪表盘 (Dashboard)

**进入路径**: 登录后默认首页,或点击左侧菜单"仪表盘"

#### 功能说明

仪表盘提供系统的关键运营数据概览,帮助运营团队快速掌握业务状态。

#### 数据指标

| 指标项 | 说明 | 数据来源 |
|--------|------|---------|
| 今日销售额 | 今日订单总金额 | 实时统计 |
| 今日订单数 | 今日新增订单数量 | 实时统计 |
| 待处理订单 | 状态为"待支付"的订单数 | 实时统计 |
| 今日核销数 | 今日核销优惠券数量 | 实时统计 |
| 商户总数 | 入驻商户总数 | 商户表统计 |
| 券模板总数 | 已创建券模板数量 | 券模板表统计 |

#### 使用建议

- 每日登录后先查看仪表盘了解整体运营状况
- 关注异常指标(如待处理订单过多)
- 作为运营决策的数据依据

**调整流程**:
1. 点击"调整库存"按钮
2. 输入调整数量(正数=增加,负数=减少)
3. 输入调整原因说明(必填,最多200字)
4. 系统实时验证调整后库存不能为负数
5. 点击"确认调整"
6. 自动记录库存变更日志(MANUAL_ADJUST)
7. 自动刷新券模板数据

#### 操作功能

**退款处理**:

对于"退款中"状态的订单,管理员可以进行审核:

1. 查看退款原因和订单信息
2. 核对订单状态(必须在有效期内且未核销)
3. 选择"同意退款"或"拒绝退款"
4. 同意退款后,系统自动调用微信退款接口
5. 退款成功后,订单状态变为"已退款"

**注意**: 退款操作不可逆,请谨慎处理

## 四、常见操作指南

### 4.1 创建券模板完整流程

1. **进入券模板管理**
   - 点击"业务管理" → "券模板管理"
   - 点击"创建券模板"按钮

2. **填写基本信息**
   - 券标题: 输入优惠券名称
   - 券模板图片: 上传封面图
   - 面值: 输入优惠券金额
   - 购买价格: 输入用户购买价格
   - 结算价格: 输入商户结算价格(可选)
   - 库存: 输入初始库存数量

3. **配置适用商户**
   - 在"适用商户范围"字段选择商户
   - 支持多选(选择多个商户)
   - 选中的商户可以核销该优惠券

4. **设置有效期和限领数量**
   - 有效期开始: 选择开始时间
   - 有效期结束: 选择结束时间
   - 每人限领数量: 输入单用户购买上限

5. **填写使用说明**
   - 输入优惠券详细使用规则
   - 例如:"满100元可用,仅限餐饮类商户"

6. **保存并上架**
   - 点击"保存"按钮
   - 在列表页点击"上架"按钮
   - 优惠券在小程序端可见

---

### 4.2 商户入驻流程

1. **进入商户管理**
   - 点击"业务管理" → "商户管理"
   - 点击"创建商户"按钮

2. **填写基本信息**
   - 商户名称: 输入商户全称
   - 商户Logo: 上传品牌Logo
   - 商户分类: 选择所属分类
   - 楼层位置: 输入楼层信息
   - 联系电话: 输入商户电话

3. **上传商户图片**
   - 上传商户环境图片(最多10张)
   - 建议上传店铺环境、商品展示等

4. **填写商户简介**
   - 输入商户详细描述
   - 特色菜品、营业时间等

5. **配置核销员**
   - 进入商户详情页 → "核销员管理"Tab
   - 点击"添加核销员"
   - 从用户列表中选择用户
   - 授权为该商户的核销员

6. **保存并激活**
   - 点击"保存"按钮
   - 商户状态默认为"已入驻"
   - 商户在小程序端可见

---

### 4.3 结算单审核流程

1. **查看待审核结算单**
   - 点击"业务管理" → "结算管理"
   - 筛选"待审核"状态的结算单

2. **进入结算详情页**
   - 点击"查看详情"
   - 查看结算金额和核销流水明细

3. **核对结算金额**
   - 检查结算金额计算是否正确
   - 公式: 本周期核销订单的结算价总和
   - 如有异常,记录异常情况

4. **查看核销流水**
   - 检查核销记录是否正常
   - 核对核销时间、商户、金额等信息
   - 排查异常核销记录

5. **审核通过**
   - 点击"审核通过"按钮
   - 填写审核备注(如有)
   - 结算状态变为"商户确认"

---

### 4.4 处理用户退款申请

1. **查看退款申请**
   - 点击"业务管理" → "订单管理"
   - 筛选"退款中"状态的订单

2. **进入订单详情页**
   - 点击"查看详情"
   - 查看退款原因和订单信息

3. **核对退款条件**
   - 订单状态为"待使用"
   - 在有效期内
   - 未被核销
   - 检查退款原因是否合理

4. **审核退款**
   - 条件符合: 点击"同意退款"
   - 条件不符: 点击"拒绝退款"
   - 填写审核备注

5. **退款执行**
   - 同意退款后,系统自动调用微信退款接口
   - 退款成功后,订单状态变为"已退款"
   - 用户收到退款通知

## 五、常见问题

### 5.1 券模板相关

#### Q1: 券模板创建后可以修改吗?

**A**: 可以。但以下字段建议谨慎修改:
- 面值和购买价格(已售出的券不变)
- 适用商户范围(可能导致已购券无法核销)
- 有效期(建议在创建时确定)

库存可以随时调整,系统会记录调整日志。

---

#### Q2: 如何调整库存?

**A**:
1. 进入券模板详情页
2. 点击"调整库存"按钮
3. 输入调整数量和原因
4. 系统记录调整日志

---

### 5.2 商户相关

#### Q3: 如何配置商户核销员?

**A**:
1. 进入商户详情页 → "核销员管理"Tab
2. 点击"添加核销员"
3. 从用户列表中选择用户
4. 用户授权后在小程序显示核销员入口

核销员只能核销适用商户范围内的优惠券。

---

### 5.3 结算相关

#### Q4: 结算金额如何计算?

**A**:
- 结算金额 = 本周期核销订单的结算价总和
- 结算价: 券模板的`settlementAmount`字段
- 如果`settlementAmount`为空,使用`faceValue`(面值)

---

### 5.4 订单相关

#### Q5: 用户退款条件是什么?

**A**:
- 订单状态为"待使用"(PAID)
- 在有效期内
- 未被核销
- 未在结算锁定周期内
